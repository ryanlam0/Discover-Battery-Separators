FILE 1: build_training_data.py
------------------------------
PURPOSE: Builds the labeled training dataset by combining multiple data sources.

WHAT IT DOES:
  Step 1: Defines 42 known separator polymers with binary safety labels
          (safe = structural integrity above 150C, unsafe = fails at or below 150C)
  Step 2: Joins with OpenPoly database to get 6 material properties
          (Tg, Tm, Td, tensile strength, Young's modulus, elongation at break)
  Step 3: Parses scraped literature tables from 5 PMC articles for separator-specific
          properties (porosity, ionic conductivity)
  Step 4: Combines all literature data and computes averages per polymer
  Step 5: Merges literature properties into training data, fills NaN with medians

OUTPUT: data/training_data.csv (39 polymers with safety labels + 8 features)

CLASS TECHNIQUES:
  - pd.read_csv()                              Lecture: Combining Data
  - pd.DataFrame() from list of dicts          Lecture: Web Scraping
  - df.merge(left_on=, right_on=, how="left")  Lecture: Types of Joins
  - df.groupby().agg()                         Lecture: Combining Data
  - pd.concat()                                Lecture: Combining Data
  - df.dropna(), df.fillna()                   General pandas
  - df.drop_duplicates()                       General pandas
  - String parsing with loops                  Lecture: Web Scraping

SLIGHTLY ADVANCED:
  - Nothing major. All standard pandas.


FILE 2: train_model.py
----------------------
PURPOSE: Trains a KNN classifier to predict polymer safety, tunes hyperparameters,
         and screens 101 untested polymers for new separator candidates.

WHAT IT DOES:
  Step 1: Loads training data, defines 6 features and safety labels
  Step 2: Builds pipeline: StandardScaler + KNeighborsClassifier
  Step 3: Cross-validation (5-fold) to evaluate baseline model
  Step 4: GridSearchCV to find best k (1-11) and metric (euclidean vs manhattan)
  Step 5: Confusion matrix and per-class precision/recall/F1 on training data
  Step 6: Screens 101 OpenPoly polymers not in training set, predicts safety,
          saves results to CSV

OUTPUTS:
  - data/screening_results.csv (101 polymers with predicted safety + probability)
  - figures/model_selection.png (F1 vs k for both metrics, best point labeled)
  - figures/screening_results.png (scatter of screened polymers colored by prediction)

CLASS TECHNIQUES:
  - make_pipeline(StandardScaler(), KNeighborsClassifier())  Lecture 11, 14
  - cross_val_score(scoring="accuracy", cv=5)                Lecture 12
  - cross_val_score(scoring="f1_macro", cv=5)                Lecture 12
  - GridSearchCV(param_grid={...}, scoring="f1_macro")       Lecture 13
  - grid_cv.best_params_, grid_cv.best_score_                Lecture 13
  - confusion_matrix(y_true, y_pred, labels=[...])           Lecture 15
  - pipeline.predict(), pipeline.predict_proba()             Lecture 14
  - pandas .plot.line(x=, y=, label=, ax=)                   Throughout class
  - px.scatter(color=, hover_data=)                          Plotly (class)

SLIGHTLY ADVANCED:
  - ax.annotate() with arrowprops for labeling best F1 point on model selection plot
  - ax.scatter() for the best point marker
  - matplotlib.use("Agg") — just saves to file instead of displaying


FILE 3: clustering_and_evaluation.py
------------------------------------
PURPOSE: K-means clustering to find natural polymer groupings, then detailed
         model evaluation with training and cross-validation confusion matrices.

WHAT IT DOES:
  Part A - Clustering:
    - Clusters all 144 polymers (with complete data) using K-means
    - Elbow plot to justify k=3
    - Visualizes clusters in Tg vs Tm space
    - Compares cluster assignments to known safety labels via crosstab
    - Overlays known safe/unsafe polymers on cluster plot

  Part B - Training Confusion Matrix:
    - Re-runs GridSearchCV to get best model
    - Confusion matrix on training data predictions
    - Per-class precision, recall, F1 table

  Part C - Cross-Validation Confusion Matrix:
    - Uses cross_val_predict to get out-of-fold predictions
    - Confusion matrix on CV predictions (more honest evaluation)
    - Per-class metrics + pooled macro F1 = 0.827

OUTPUTS:
  - figures/elbow_plot.png (inertia vs k, shows elbow at k=3)
  - figures/kmeans_clusters.png (clusters in Tg vs Tm space)
  - figures/clusters_vs_safety.png (clusters + known safe/unsafe overlaid)
  - figures/confusion_matrix.png (training data confusion matrix)
  - figures/confusion_matrix_cv.png (cross-validation confusion matrix — USE THIS ONE)

CLASS TECHNIQUES:
  - make_pipeline(StandardScaler(), KMeans(n_clusters=k))    Lecture 16
  - KMeans .labels_, .inertia_                               Lecture 16
  - Elbow plot with pandas .plot.line()                      Lecture 16
  - pd.crosstab()                                            General pandas
  - GridSearchCV                                             Lecture 13
  - confusion_matrix()                                       Lecture 15
  - precision_score(), recall_score(), f1_score()            Lecture 15
  - px.scatter(color=, hover_data=)                          Plotly (class)
  - df.merge(left_on=, right_on=, how="left")                Lecture: Joins

SLIGHTLY ADVANCED:
  - cross_val_predict() — same concept as cross_val_score but returns predictions
    instead of scores. Easy to explain: "it predicts each polymer using a model
    that was NOT trained on that polymer"
  - ConfusionMatrixDisplay() — same pattern as PrecisionRecallDisplay from Lecture 15
  - fig.add_scatter() in plotly — adds extra markers on top of a plotly plot
  - ax.table() — puts a table of numbers next to the confusion matrix heatmap
  - plt.subplots(1, 2, gridspec_kw={"width_ratios": [1, 0.8]}) — side-by-side layout


FILE 4: plot_candidates.py
--------------------------
PURPOSE: Visualizes the top predicted safe separator candidates.

WHAT IT DOES:
  Plot 1: Scatter of all candidates in Tg vs Tm space
    - Known safe (green), known unsafe (red) from training data
    - Top candidates at 100% safe probability (dark blue)
    - Moderate candidates at 50-99% (light blue)
    - Labels the most interesting candidates with names

  Plot 2: Bar chart of the 5 polymers predicted 100% safe
    - Shows all 6 material properties side by side
    - Only shows properties that actually vary (not imputed medians)

OUTPUTS:
  - figures/top_candidates.png (labeled scatter of top candidates)
  - figures/candidate_properties.png (bar chart of 100% safe candidate properties)

CLASS TECHNIQUES:
  - px.scatter(color=, hover_data=, color_discrete_map=)     Plotly (class)
  - fig.add_annotation() for labeling points                 Plotly extension
  - pandas .plot.barh(ax=)                                   Throughout class
  - df.drop_duplicates(), df.sort_values()                   General pandas

SLIGHTLY ADVANCED:
  - fig.add_annotation() with custom offsets per label — plotly annotations
  - plt.subplots(2, 3) for the 6-panel bar chart grid
